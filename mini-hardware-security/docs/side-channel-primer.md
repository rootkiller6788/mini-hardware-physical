# 侧信道攻击入门 (Side-Channel Attack Primer)

> 侧信道攻击利用物理实现的信息泄露来获取秘密，而非直接攻击算法本身

---

## 1. 什么是侧信道攻击

侧信道攻击 (Side-Channel Attack) 是利用计算机系统的物理实现而非算法漏洞来提取信息的一类攻击方式。
任何物理可测量的量——时序、功耗、电磁辐射、声音等——都可能成为信息泄露的渠道。

### 侧信道分类

| 侧信道类型 | 测量量 | 精度 | 威胁距离 |
|-----------|-------|------|---------|
| 时序 (Timing) | 执行时间 | 纳秒级 | 远程/本地 |
| 功耗 (Power) | 电流消耗 | 毫瓦级 | 近距离 |
| 电磁 (EM) | 电磁辐射 | 微伏级 | 中距离 |
| 声学 (Acoustic) | 声音/振动 | 分贝级 | 中距离 |
| 热学 (Thermal) | 温度 | 摄氏度级 | 近距离 |
| 光子 (Photon) | 芯片发光 | 光子计数 | 近距离 |

## 2. 时序攻击 (Timing Attacks)

### 2.1 基本原量

操作执行时间依赖输入数据 -> 攻击者测量时间推断秘密信息

### 2.2 缓存时序攻击

```
秘密数据 -> 影响内存访问模式 -> 影响缓存命中/未命中 -> 影响执行时间 -> 攻击者推断

缓存命中：L1 访问 ~4ns
缓存未命中：DRAM 访问 ~100ns
差异：25x 时间差可被准确测量
```

### 2.3 代码示例：泄漏的字符串比较

```c
// 不安全：逐字符比较，返回时间泄露匹配失败位置
int strcmp_unsafe(const char *a, const char *b) {
    while (*a && *b && *a == *b) {  // <-- 早退导致时间泄露!
        a++; b++;
    }
    return *a - *b;
}

// 安全：常数时间比较
int strcmp_safe(const char *a, const char *b, int len) {
    int result = 0;
    for (int i = 0; i < len; i++) {
        result |= (a[i] ^ b[i]);  // 始终执行所有比较
    }
    return result;
}
```

## 3. 功耗分析 (Power Analysis)

### 3.1 简单功耗分析 (SPA)

直接观察功耗轨迹，推断操作和操作数：

```
时间 →

功耗 │  │\     /│\     /│         │
     │  │ \   / │ \   / │         │
     │  │  \ /  │  \ /  │         │
     │__│___X___│___X___│_________│___
     
     AES 轮密钥加 -> S-Box -> MixColumns -> ...

SPA 能直接看到：
  - 加密轮数
  - 乘法 vs 加法
  - 密钥操作 vs 数据操作
```

### 3.2 差分功耗分析 (DPA)

DPA 使用统计学方法从大量功耗轨迹中提取秘密信息：

```
步骤 1: 收集 N 条功耗轨迹 T[i] (已知明文/密文)
步骤 2: 猜测密钥片段 K_guess
步骤 3: 对每个 K_guess 计算内部状态
步骤 4: 按状态值将轨迹分为两组 (0-组 和 1-组)
步骤 5: 计算差分轨迹 = 平均值(1-组) - 平均值(0-组)
步骤 6: 如果 K_guess 错误 -> 差分轨迹平缓
步骤 7: 如果 K_guess 正确 -> 差分轨迹在相关时刻出现尖峰 (spike)
```

### 3.3 功耗模型

| 模型 | 公式 | 精度 | 复杂度 |
|-----|------|------|-------|
| Hamming Weight | 功耗 ∝ HW(值) | 低 | 最低 |
| Hamming Distance | 功耗 ∝ HD(旧值, 新值) | 中 | 中 |
| 比特翻转模型 | 功耗 ∝ #bits_flipped | 中 | 中 |
| 开关活动模型 | 详细电路建模 | 高 | 最高 |

```c
// Hamming Weight 模型
int hw = __builtin_popcount(value);      // 0-8
int power = hw * POWER_PER_BIT;          // 简化的线性模型
```

## 4. 电磁攻击 (EM Attacks)

### 4.1 原理

处理器在执行指令时，电流流过导线产生电磁场。不同操作产生不同的 EM 发射。

### 4.2 优势

- 不需要物理连接到芯片（无接触）
- 可聚焦到芯片特定区域（高空间分辨率）
- 可穿透封裝

### 4.3 与功耗分析的比较

| 特性 | 功耗分析 | EM 分析 |
|-----|---------|--------|
| 信噪比 | 中 | 高 (可局部聚焦) |
| 设备需求 | 分流电阻/探针 | EM 近场探针 |
| 空间分辨率 | 全局 | 局部 (mm级) |
| 攻击难度 | 低-中 | 中-高 |

## 5. 声学攻击 (Acoustic Attacks)

### 5.1 原理

电容器和电感器的振动（电致伸缩效应）产生声音，频率依赖于电流变化，
与计算活动相关。

### 5.2 GPG 密钥提取 (Shamir, 2013)

- 通过录音提取 RSA 解密中的密钥位
- 不同的操作（模乘 vs 模平方）产生不同的声学信号
- 可从 4 米外通过抛物面麦克风攻击

## 6. 对抗措施 (Countermeasures)

### 6.1 掩码 (Masking)

将秘密值分解为多个随机分享：

```c
// 一阶掩码
uint8_t secret = 0x3A;
uint8_t mask = random_byte();           // 随机掩码
uint8_t s_masked = secret ^ mask;       // 秘密与掩码异或
uint8_t s_prime = s_masked ^ mask;      // 还原: s' = (secret ^ mask) ^ mask = secret
```

### 6.2 隐藏 (Hiding)

使功耗曲线"平坦化"：

```c
// 互补逻辑: 计算秘密值时同时计算其补值
int compute_balanced(int a, int b) {
    int result = a ^ b;                  // 正常计算
    int complement = ~(a ^ b);           // 补值计算（功耗抵消）
    return result;
}
```

### 6.3 平衡 (Balancing)

- 双轨逻辑 (Dual-Rail Logic): 每比特用两条线表示
- 功耗与数据无关

### 6.4 随机化 (Randomization)

- 随机化操作顺序
- 随机化执行时间
- 添加虚拟操作

### 6.5 硬件防御

| 防御技术 | 针对 | 效果 |
|---------|------|------|
| 屏蔽层 (Shielding) | EM攻击 | 高 |
| 时钟抖动 | 时序/SPA | 中 |
| 片上电压调节 | DPA | 中 |
| 安全销毁 | 物理攻击 | 高 |
